📋 Temel Yapı

Her HTML belgesinin temelini oluşturan etiketler. Bunlar olmadan web sayfası çalışmaz!

<!DOCTYPE html>
Web tarayıcıya "Bu bir HTML5 belgesidir" demektir. Her HTML dosyasının en başında bulunması ZORUNLUDUR.

Kullanım Yeri:

Her HTML dosyasının ilk satırı! Dosyayı HTML olarak tanınmasını sağlar.
<!DOCTYPE html> <html> <head>...</head> <body>...</body> </html>
<html>
Bütün HTML içeriğini kapsayan en üst seviyedeki etiket. Web sayfasının anahtarı!

Önemli Öznitelikleri:

  • lang - Sayfanın dilini belirtir. "tr" = Türkçe, "en" = İngilizce
<html lang="tr"> </html>
♿ Erişilebilirlik İpucu: lang özniteliğini eklemek ekran okuyucuların doğru dili seçmesini sağlar.
<head>
Web sayfasının "bilgi merkezi"dir. Başlık, açıklama, bağlantılar ve meta bilgileri buraya gider. Tarayıcı ve arama motorları buradaki bilgileri kullanır.

İçinde Bulunabilecekler:

  • <title> - Tarayıcı sekmesinde görünen başlık
  • <meta> - Sayfanın açıklaması, charset (karakter seti) vs.
  • <link> - CSS dosyaları, favicon vs.
  • <script> - JavaScript kodları
  • <style> - CSS stilleri
<head> <meta charset="UTF-8"> <title>Benim Harika Websitem</title> <link rel="stylesheet" href="styles.css"> </head>
<body>
Web sayfasında kullanıcının görebileceği BÜTÜNa içeriğin bulunduğu konteyner. Metin, resimler, linkler, formlar vb. hepsi buraya gider.
<body> <h1>Hoş Geldiniz!</h1> <p>Bu metni herkes görebilir.</p> </body>
<meta charset="UTF-8">
Sayfanın karakter setini belirtir. UTF-8, dünyadaki TÜM harfleri (ş, ç, ğ, ı vb.) destekler. Türkçe yazı yazacaksanız ÖNCELİKLE bu satırı ekleyin!
Çok Önemli: UTF-8 olmadan Türkçe karakterler yanlış gösterilir!
<meta charset="UTF-8">
<title>
Tarayıcı sekmesinde ve arama motoru sonuçlarında görünen başlık. İlk izlenim burada başlar!

Ne Zaman Kullanılır:

Her sayfanın kendine özgü, açıklayıcı bir başlığı olmalıdır. SEO için çok önemlidir.
<title>Benim Tasarım Portföyüm | İbrahim Yilmaz</title>
<link>
Sayfaya dış dosyaları (CSS, favicon vb.) bağlamak için kullanılır. Sayfanın görünümünü kontrol eden CSS dosyaları genellikle burada bağlanır.

Önemli Öznitelikleri:

  • rel - Dosyanın ilişkisi ("stylesheet" = CSS dosyası)
  • href - Dosyanın konumu
  • type - Dosyanın tipi (genellikle "text/css")
<link rel="stylesheet" href="styles.css"> <link rel="icon" href="favicon.ico">
<script>
JavaScript kodunu sayfaya eklemek için kullanılır. Butonlara tıklama işlevselliği, animasyonlar vb. hepsi JavaScript ile yapılır.

Önemli Öznitelikleri:

  • src - Dış JavaScript dosyasının konumu
  • type - Genellikle "text/javascript"
  • async - Dosya indirme sırasında sayfanın yüklenmesini durdurmaması
  • defer - Sayfanın yüklenmesinden sonra script çalış
<!-- Dış JavaScript dosyası --> <script src="script.js" defer></script> <!-- Direkt JavaScript kodu --> <script> console.log('Merhaba Dünya!'); </script>

📝 Metin Etiketleri

Metni biçimlendirmek ve anlamlandırmak için kullanılan etiketler.

<h1> ... <h6>
Başlıklar! h1 en büyük ve en önemli, h6 en küçük ve en az önemli. Bir sayfada genellikle sadece BİR tane h1 olmalıdır.
SEO İpucu: Başlıklar arama motorları tarafından çok önemli görülür. Ana başlık için h1 kullanın!
<h1>Sayfa Başlığı (En Büyük)</h1> <h2>Alt Başlık</h2> <h3>Daha Küçük Başlık</h3>
♿ Erişilebilirlik: Başlıklar sayfanın yapısını gösterir. Ekran okuyucular başlıkları takip ederek içeriği gezinir.
<p>
Paragraf (paragraph). Metin bloklarını ayrı tutmak için kullanılır. Normal sayfada gördüğünüz metin bloklarının neredeyse hepsi p etiketidir.
<p>Bu bir paragraf. Paragraflar otomatik olarak metin bloklarını ayırır.</p> <p>İkinci paragraf. Arasında boşluk vardır.</p>
<span>
Küçük metin parçalarını biçimlendirilmek için kullanılır. Paragraflı içinde belirli kelimeleri vurgulama için ideal.
<p>Bu paragrafta <span style="color: red;">bu kısım kırmızı</span> gösterilir.</p>
<strong> & <em>
<strong> - Metni BOLD (kalın) yapar ve SEO açısından önemli gösterir.
<em> - Metni İTALİC (eğik) yapar ve vurgu anlamı taşır.
<p>Bu <strong>çok önemli</strong> ve bu <em>vurgulu</em>.</p>
<br>
Satır sonu (break). Metni yeni satıra geçirmek için kullanılır. UYARI: Layout kontrol etmek için br kullanmayın, bunun için CSS kullanın!
Adres:
Ankara Caddesi No: 123
Ankara, Türkiye
<hr>
Yatay çizgi (horizontal rule). İçeriği bölümlere ayırmak için kullanılır. Görsel ayırıcı.
<p>Birinci bölüm</p> <hr> <p>İkinci bölüm</p>
<blockquote>
Alıntı yapmak için kullanılır. Başka birinin sözünü alıntıladığınızda bu etiketj kullanın.
<blockquote> <p>İyi tasarım, kullanıcı deneyiminin temelini oluşturur. - Steve Jobs</p> </blockquote>
<pre> & <code>
<pre> - Metni olduğu gibi gösterir (boşluklar, satır sonları korunur).
<code> - Kod parçalarını gösterir (Courier New yazı tipi).
<pre><code> function hello() { console.log('Merhaba'); } </code></pre>
<mark>
Metin vurgula/highlight yap. Sayfa içinde arama sonucunda bulunan kelimeleri vurgulamak için ideal.
<p>HTML öğrenmek <mark>çok önemli</mark> bir beceridir.</p>
<small>, <sub>, <sup>
<small> - Metni küçültür (yaşlı insanlar için de iyidir).
<sub> - Alt yazı (H₂O gibi).
<sup> - Üst yazı (x² gibi).
<p>Su H<sub>2</sub>O'dur. E=mc<sup>2</sup></p> <p><small>Bu çok küçük yazı</small></p>

🔗 Bağlantılar & Görseller

Web'i web yapan bağlantılar ve sayfaları güzel gösteren görseller.

<a>
Bağlantı (anchor). Başka sayfaya veya web sitesine gitmek için tıklanabilir metin yaratır. Web'in kalbi!

Önemli Öznitelikleri:

  • href - Gidilecek sayfanın adresi (ZORUNLU)
  • target - "_blank" = yeni sekmede aç, "_self" = aynı sekmede aç (varsayılan)
  • rel - "nofollow" = arama motorları bunu takip etmesin, "external" = dış bağlantı
  • title - Fare üzerine getirilince görünen açıklama
<a href="https://google.com" target="_blank">Google'a Git</a> <a href="/hakkimizda.html">Hakkımızda</a> <a href="#bölüm2">Bölüm 2'ye Git</a>
♿ Erişilebilirlik: Bağlantı metni açıklayıcı olmalıdır. "Tıkla" yerine "Google'a Git" yazın.
<img>
Resim gösterir. Web sayfalarını güzel yapan önemli eleman! Self-closing yani kapatma etiketi yoktur.

Önemli Öznitelikleri:

  • src - Resim dosyasının yolu (ZORUNLU)
  • alt - Resim yüklenemediğinde gösterilen metin (ZORUNLU - SEO ve erişilebilirlik için)
  • width - Resimin genişliği (piksel veya %)
  • height - Resimin yüksekliği
  • loading - "lazy" = sayfaya yaklaşınca yükle (hız için iyi)
  • srcset - Farklı ekran boyutları için farklı resimler
<img src="resimler/logo.png" alt="Şirket Logosu" width="200"> <img src="foto.jpg" alt="Dağ Manzarası" loading="lazy">
Alt Metni Çok Önemlidir! Görme engellililer için resmin ne olduğunu açıklar. SEO açısından da önemlidir.
♿ Erişilebilirlik: Alt metni resmin içeriğini açıklamalıdır. "Resim" yerine "Sabahlayan Dağ Manzarası" yazın.
<picture> & <source>
Farklı ekran boyutları için farklı resimler göstermek için kullanılır. Mobil telefonlarda küçük resim, bilgisayarda büyük resim gibi.
<picture> <source media="(max-width: 600px)" srcset="resim-kucuk.jpg"> <source media="(min-width: 601px)" srcset="resim-buyuk.jpg"> <img src="resim-varsayilan.jpg" alt="Açıklama"> </picture>

📋 Listeler

Öğeleri organize etmek için harika etiketler.

<ul>
Sırasız liste (unordered list). Satın alınacak ürünler, özellikler vb. için kullanılır. Her öğe bullet (•) ile başlar.
<ul> <li>HTML öğrenin</li> <li>CSS öğrenin</li> <li>JavaScript öğrenin</li> </ul>
<ol>
Sıralı liste (ordered list). Adım adım yapılması gereken şeyler için kullanılır. Otomatik olarak numaralandırılır (1, 2, 3...).

Önemli Öznitelikleri:

  • type - "1" (sayılar), "a" (küçük harfler), "A" (büyük harfler), "i" (romen rakamları)
  • start - Kaçtan başlanacağı (varsayılan: 1)
  • reversed - Ters sırada (10, 9, 8...)
<ol> <li>Malzemeleri hazırla</li> <li>Karıştır</li> <li>Fırınla</li> </ol> <!-- Romen rakamları ile --> <ol type="i"> <li>Birinci</li> <li>İkinci</li> </ol>
<li>
Liste öğesi (list item). Listedeki her öğe bu etiketle çerçevelenmelidir.
<dl>, <dt>, <dd>
<dl> - Tanım listesi (definition list).
<dt> - Tanım terimi (definition term).
<dd> - Tanım açıklaması (definition description).
Sözlük, terim açıklaması vb. için kullanılır.
<dl> <dt>HTML</dt> <dd>Web sayfasının yapısını belirtir</dd> <dt>CSS</dt> <dd>Web sayfasının görünümünü kontrol eder</dd> </dl>

📊 Tablolar

Verileri satır ve sütunlarda organize etmek için.

<table>
Tablo oluşturur. İçinde thead, tbody, tfoot ve tr, th, td etiketleri bulunur.
<table> <thead> <tr> <th>Ürün</th> <th>Fiyat</th> </tr> </thead> <tbody> <tr> <td>Kitap</td> <td>₺50</td> </tr> </tbody> </table>
<thead>, <tbody>, <tfoot>
<thead> - Tablo başlığı (başlık satırı).
<tbody> - Tablo gövdesi (ana veri).
<tfoot> - Tablo ayağı (özet/toplam satırı).
<tr>, <th>, <td>
<tr> - Tablo satırı (table row).
<th> - Tablo başlık hücresi (table header). Varsayılan olarak BOLD ve merkez hizalıdır.
<td> - Tablo veri hücresi (table data).

Önemli Öznitelikleri (td & th için):

  • colspan - Hücrenin kaç sütun boyutunda olacağı
  • rowspan - Hücrenin kaç satır boyutunda olacağı
<table> <tr> <th>Adı</th> <th colspan="2">İletişim</th> </tr> <tr> <td>Ahmet</td> <td>ahmet@email.com</td> <td>05551234567</td> </tr> </table>

📝 Formlar

Kullanıcıdan veri almak için - arama kutuları, giriş sayfaları vb.

<form>
Form oluşturur. İçinde input, textarea, select vb. alanlar bulunur. Verileri sunucuya gönderir.

Önemli Öznitelikleri:

  • action - Form verilerinin gönderileceği sunucu adresi
  • method - "GET" (URL'de görünür) veya "POST" (gizli gönderim)
  • enctype - Dosya yükleme için "multipart/form-data"
<form action="/gonder" method="POST"> <!-- Form alanları buraya --> </form>
<input>
Kullanıcıdan veri alan alan. Türe bağlı olarak farklı şekillerde görünür. Self-closing etiket!

Türleri (type özniteliği):

  • text - Sade metin (adı, soyadı vb.)
  • password - Şifre (yazılanlar • • • şeklinde gösterilir)
  • email - E-posta (@ kontrolü yapar)
  • number - Sayı (sadece rakamlar kabul eder)
  • tel - Telefon numarası
  • url - Web adresi
  • date - Tarih seçme takvimi
  • time - Saat seçme
  • color - Renk seçme palesiz
  • range - Kaydırıcı (1-100 arası seç gibi)
  • checkbox - Onay kutusu (birden fazla seç)
  • radio - Radyo düğmesi (birini seç)
  • file - Dosya yükleme
  • submit - Gönder düğmesi
  • reset - Sıfırla düğmesi
  • button - Normal düğme
  • hidden - Gizli alan (kullanıcı görmez)
  • search - Arama kutusu

Ortak Öznitelikleri:

  • name - Alan adı (form sunucuya gönderirken bu adı kullanır)
  • value - Varsayılan değer
  • placeholder - Alan boşken gösterilen ipucu metni
  • required - Bu alan zorunlu (doldurulması gerekir)
  • disabled - Alan devre dışı (tıklanamaz)
  • readonly - Alan okunabilir ancak değiştirilmez
  • maxlength - En fazla kaç karakter yazılabileceği
  • min & max - Minimum ve maksimum değer (number, range için)
<input type="text" placeholder="Adınızı girin"> <input type="email" required> <input type="password" placeholder="Şifreni girin"> <input type="checkbox"> Şartları Kabul Ediyorum <input type="submit" value="Gönder">
<textarea>
Çok satırlı metin alanı. Yorum, mesaj vb. için ideal. input'tan farkı: birden fazla satır yazılabilir.

Önemli Öznitelikleri:

  • rows - Kaç satır yüksekliğinde olacağı
  • cols - Kaç karakterlik genişlik
  • placeholder - İpucu metni
  • maxlength - En fazla kaç karakter
<textarea rows="5" cols="40" placeholder="Mesajınızı yazın..."></textarea>
<select> & <option>
Açılır menü (dropdown). Seçenekler arasında seçim yapılır. Kontrol edilen seçimler için ideal.
<select name="renk"> <option>Renk Seçin</option> <option value="kirmizi">Kırmızı</option> <option value="mavi">Mavi</option> <option value="yesil">Yeşil</option> </select>
<label>
Form alanları için etiket. Hangi alanın ne olduğunu açıklar. Erişilebilirlik için ÇOK ÖNEMLİ!
<label for="email-input">E-posta:</label> <input type="email" id="email-input">
♿ Erişilebilirlik: Label'in "for" özniteliği input'un "id" ile eşleşmelidir.
<fieldset> & <legend>
Form alanlarını gruplandırır. Legend başlık olur. Ilgili alanları organize eder.
<fieldset> <legend>Kişisel Bilgiler</legend> <label>Ad: <input type="text"></label> <label>Soyad: <input type="text"></label> </fieldset>
<button>
Tıklanabilir düğme. Input submit'in aksine, button içinde HTML (resim, metin vb.) konulabilir.

Önemli Öznitelikleri:

  • type - "submit" (form gönder), "reset" (sıfırla), "button" (normal)
  • disabled - Düğmeyi devre dışı bırak
<button type="submit">Formu Gönder</button> <button type="button">Beni Tıkla!</button>
<datalist>
Input alanı için öneriler listesi. Yazarken uygun öneriler gösterilir. Google arama gibi!
<input type="text" list="ülkeler" placeholder="Ülke yazın"> <datalist id="ülkeler"> <option>Türkiye</option> <option>Amerika</option> <option>İngiltere</option> </datalist>

🏗️ Semantik HTML5

Sayfanın yapısını anlatan etiketler. Arama motorları ve ekran okuyucuları için çok önemli!

<header>
Sayfa veya bölümün başlığı. Logo, navigasyon, arama kutusu vb. burada olur.
<header> <img src="logo.png" alt="Logo"> <nav>...</nav> </header>
<nav>
Navigasyon (menü). Siteyi dolaşmak için kullanılan bağlantılar burada olur.
<nav> <ul> <li><a href="/">Ana Sayfa</a></li> <li><a href="/hakkimizda">Hakkımızda</a></li> </ul> </nav>
<main>
Sayfanın ana içeriği. Her sayfada sadece BİR tane main olmalıdır. Header, footer, sidebar vb. içermez.
<article>
Bağımsız içerik. Blog yazısı, haber, ürün açıklaması vb. Her article kendi başına anlamlı olmalıdır.
<section>
Mantıksal bölüm. İlgili içeriği gruplaşımak için kullanılır. Başlık içermelidir.
<aside>
Yan taraf içeriği. Sidebar, ilgili linkler, reklam vb. Main içerikten ayrı şeyler.
<footer>
Sayfanın altı. Telif hakkı, linkler, iletişim bilgileri vb. burada olur.
<footer> <p>© 2026 Benim Şirketim. Tüm hakları saklıdır.</p> </footer>
<figure> & <figcaption>
Resim veya şemayı başlığı ile birlikte gösterir. Şekil + açıklama için ideal.
<figure> <img src="grafik.png" alt="Satış Grafiği"> <figcaption>2026 Yılı Satış Rakamları</figcaption> </figure>
<details> & <summary>
Genişletilir/daraltılabilir içerik. S.S.S. (Sıkça Sorulan Sorular) için ideal. JavaScript gerekmez!
<details> <summary>HTML nedir?</summary> <p>HTML, web sayfasının yapısını belirtir...</p> </details>
<dialog>
Modal dialog kutusu. Uyarı, onay vb. için kullanılır. showModal() ile açılır.
<time>
Tarih veya zaman belirtir. Arama motorları bu bilgiyi anlayabilir.
<p>Makale <time datetime="2026-03-27">27 Mart 2026</time> tarihinde yayınlandı.</p>

🎬 Medya

Video, ses, ve etkileşimli içerik için etiketler.

<video>
Video gösterir. YouTube embed etmek yerine kendi videolarınızı barındırabileceğiniz alternatif.

Önemli Öznitelikleri:

  • src - Video dosyasının yolu
  • controls - Oynat/duraklat/ses düğmelerini göster
  • autoplay - Sayfaya girince otomatik oynat
  • loop - Bitiş sonra tekrar başla
  • muted - Sessiz başla (autoplay için gerekli)
  • poster - Oynatmadan önce gösterilecek resim
  • width & height - Video boyutu
<video width="320" height="240" controls> <source src="video.mp4" type="video/mp4"> Tarayıcınız video desteklemiyor </video>
<audio>
Ses oynatır. Müzik, podcast vb. için kullanılır.
<audio controls> <source src="muzik.mp3" type="audio/mpeg"> Tarayıcınız ses desteklemiyor </audio>
<iframe>
Başka bir web sayfasını veya video (YouTube vb.) sayfanın içine gömütür. Inline frame = sayfanın içindeki frame.

Önemli Öznitelikleri:

  • src - Gömülecek sayfanın URL'si
  • width & height - Boyutlar
  • frameborder - Çerçeve göster (1) veya gösterme (0)
  • allow - İzinler (fullscreen, camera vb.)
<!-- YouTube Videosu --> <iframe width="560" height="315" src="https://www.youtube.com/embed/..." allow="fullscreen"> </iframe>
<canvas>
Boş bir tuvale JavaScript ile şekil, grafik vb. çizebileceğiniz alan. Oyunlar, animasyonlar vb. için kullanılır.
<canvas id="myCanvas" width="400" height="400"></canvas> <script> const canvas = document.getElementById('myCanvas'); const ctx = canvas.getContext('2d'); ctx.fillStyle = 'red'; ctx.fillRect(0, 0, 100, 100); </script>
<svg>
Vektör grafikleri (ölçeklendirilebilir, keskin kalır). İkonlar, loggolar vb. için ideal.
<svg width="100" height="100"> <circle cx="50" cy="50" r="40" fill="blue"></circle> </svg>

🔍 Meta & SEO

Arama motorları ve sosyal medya için önemli bilgiler.

<meta name="description">
Google'daki arama sonuçlarında görünen açıklamadan. 155-160 karakter olmalı.
<meta name="description" content="UI/UX Tasarımcıları için HTML öğrenme rehberi. Yazılım bilgisi gerekmez!">
<meta name="viewport">
Mobil cihazlarda sayfanın doğru görünmesini sağlar. Responsive tasarım için ZORUNLU!
<meta name="viewport" content="width=device-width, initial-scale=1.0">
Open Graph Tags (<meta property="og:...">)
Sayfayı Twitter, Facebook vb. sosyal ağlarda paylaşırken görünen bilgiler. Instagram'da güzel görünmek için önemli!
<meta property="og:title" content="HTML Cheatsheet"> <meta property="og:description" content="UI/UX Tasarımcıları için eksiksiz rehber"> <meta property="og:image" content="https://example.com/og-image.jpg"> <meta property="og:url" content="https://example.com">
<link rel="canonical">
Ana sayfayı belirtir. Aynı içeriğin birden fazla URL'si varsa ("?page=1" vs) hangisinin ana olduğunu söyle.
<link rel="canonical" href="https://example.com/sayfam">
<meta name="robots">
Google'a sayfayı index etmesini mi yoksa görmezden gelmesini mi söyler.
<meta name="robots" content="index, follow"> <!-- "index, follow" = normal (varsayılan) --> <!-- "noindex, nofollow" = gizli sayfa -->

♿ Erişilebilirlik (Accessibility)

Engelli insanların da sayfanızı kullanabilmesi için önemli öznitelikleri.

aria-label
Ekran okuyucuya element'in ne olduğunu söyler. Örneğin resim olmadan bir buton varsa neyi yaptığını açıklar.
<button aria-label="Menüyü Aç">☰</button>
aria-describedby
Element'in ayrıntılı açıklamasını başka bir elemento referans eder.
<input type="password" aria-describedby="password-hint"> <p id="password-hint">En az 8 karakter</p>
aria-hidden="true"
Element'i ekran okuyuculardan gizler. Dekoratif öğeleri (sadece görünüş için) gizlemek için kullanılır.
<span aria-hidden="true">★★★★★</span> 5 Yıldız
tabindex
Klavye ile Tab tuşu basılırken elementin sırasını belirtir. Sağlam navigasyon için önemli.
<button tabindex="1">Birinci Buton</button> <button tabindex="2">İkinci Buton</button>
role
Element'in ne işe yaradığını açıkça söyler. Örneğin div'i button gibi davranması için role="button".
<div role="button" tabindex="0">Tıkla Beni</div>

🌍 Global Öznitelikler

TÜM HTML etiketlerinde kullanılabilecek ortak öznitelikler.

class
CSS ve JavaScript tarafından element'i seçmek için kullanılır. Birden fazla class alanı olabilir (space ile ayrılır).
<div class="container header-section">...</div>
id
Sayfada benzersiz (unique) tanımlayıcı. Sayfa içindeki bağlantılarda ve CSS'de kullanılır. Bir id sadece BİR elemento verilmelidir!
<h2 id="bölüm-1">Bölüm 1</h2> <a href="#bölüm-1">Bölüm 1'e Git</a>
style
Element'e direkt CSS yazma. Genellikle externnal CSS dosyaları tercih edilir ama hızlı teste yarar.
<p style="color: red; font-size: 18px;">Kırmızı metin</p>
data-*
Kişiye özel veri saklama özniteliği. JavaScript tarafından kolayca erişilebilir. Örneğin data-user-id="123".
<button data-product-id="42" data-price="99.99">Satın Al</button>
hidden
Element'i gizler. CSS display: none ile aynı işe yarar ama daha semantik.
<p hidden>Bu paragraf gizlidir</p>
title
Element'e fare üzerine getirilince gösterilen tooltip. Kullanıcıya ipucu verir.
<button title="Sayfayı yenile">🔄</button>
lang
Element'in dilini belirtir. tr = Türkçe, en = İngilizce vb. Ekran okuyucular bunu kullanır.
<p lang="en">Hello World</p>
dir
Metnin yönünü belirtir. ltr = soldan sağa (İngilizce), rtl = sağdan sola (Arapça, Farsça vb.).
<p dir="rtl">مرحبا بالعالم</p>

💡 İpuçları

1. Semantik HTML Kullanın: Divlerin yerine header, nav, article vb. etiketleri kullanın. Arama motorları ve ekran okuyucuları bunları anlayabilir.
2. Her Resime Alt Metni Ekleyin: Görme engelliler için ve SEO açısından çok önemlidir.
3. Meta Öznitelikleri Unutmayın: charset, viewport, description vb. her sayfada bulunmalıdır.
4. Erişilebilirlik Sağlayın: ARIA etiketlerini, tabindex'i ve semantik HTML'i kullanarak herkesin kullanabileceği sayfalar yapın.
5. Validasyon Yapın: W3C Validator'ü kullanarak HTML kodunuzun doğru olduğunu kontrol edin.
