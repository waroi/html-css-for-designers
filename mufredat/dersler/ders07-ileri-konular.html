1. CSS Değişkenleri (Custom Properties)

CSS özel özellikleri (custom properties), CSS kodunuzda değerleri yeniden kullanılabilir değişkenler olarak tanımlamanıza izin verir. Bu sayede tema yönetimi, bakım ve tutarlılık çok daha kolay hale gelir.

Neden CSS Değişkenleri Kullanmalıyız?

  • Renk ve boyut değerlerini merkezi olarak yönetme
  • Tema değişiklikleri çok daha hızlı yapma
  • JavaScript ile dinamik olarak değerleri değiştirme
  • Kod tekrarını azaltma
  • Bakım ve güncellemeyi kolaylaştırma

CSS Kodu

:root {
  --primary: #6366f1;
  --success: #10b981;
  --spacing: 1rem;
  --border-radius: 0.5rem;
}

.button {
  background: var(--primary);
  padding: var(--spacing);
  border-radius: var(--border-radius);
}

.button.success {
  background: var(--success);
}

Sonuç

Dinamik Tema Değişimi

JavaScript

function setThemeColor(color) {
  document.documentElement.style
    .setProperty('--primary', color);
}

// Kullanım
setThemeColor('#ec4899');

Açıklama

Dinamik olarak tema rengi değiştirebilirsiniz:

💡 İpucu: CSS değişkenleri Cascade'i takip eder. Spesifik seçicilerde değişkenleri tanımlayarak lokal tema değişimleri yapabilirsiniz.

2. BEM Metodolojisi (Block, Element, Modifier)

BEM, CSS sınıflarını adlandırırken kullanılan bir metodoloji olup, kodunuzu daha organize ve bakım yapılabilir hale getirir.

BEM Yapısı

  • Block: Bağımsız bir bileşen (örn: card, button, header)
  • Element: Bloğun parçası, kendi başına anlamsız (örn: card__title, card__description)
  • Modifier: Bloğun veya Elemanın durumunu değiştirir (örn: card--featured, button--disabled)

BEM Örnekleri

Naming Convention:

.card { }              /* Block */
.card__title { }        /* Element */
.card__description { }  /* Element */
.card--featured { }     /* Modifier */
.card__title--large { } /* Element Modifier */

3. Erişilebilirlik (Accessibility)

Web erişilebilirliği, engelli kişilerin web sitelerinizi etkili bir şekilde kullanmasını sağlamak anlamına gelir. Bu bir hak ve iyi bir uygulamadır.

Erişilebilirlik İlkeleri

  • Algılanabilir (Perceivable): Kullanıcılar içeriği algılayabilmelidir
  • Kullanılabilir (Operable): Kullanıcılar arayüzü kullanabilmelidir
  • Anlaşılabilir (Understandable): İçerik ve işlemler açık olmalıdır
  • Güçlü (Robust): Çeşitli cihazlar ve teknolojilerle uyumlu olmalıdır

Erişilebilir Form Örneği

Erişilebilirlik Kontrol Listesi

✓ Uygun HTML semantik elemanlar kullanın (header, nav, main, article, footer)
✓ Form etiketleri her input için belirtilmiş olmalıdır
✓ Alternatif metin (alt) tüm anlamlı görseller için belirtilmiş olmalıdır
✓ Renk tek başına bilgi iletmemelidir (kör renk ayırt etme)
✓ Yeterli kontrast oranı (WCAG AA için 4.5:1 yazı için)
✓ Klavye navigasyonuna izin verin (Tab, Enter, vb.)
🎯 Alıştırma: Sayfanızın WAVE Web Accessibility Evaluation Tool (https://wave.webaim.org/) ile kontrol edin ve hataları düzeltin.

4. SEO Optimizasyonu

Arama Motoru Optimizasyonu (SEO), web sitenizin Google, Bing gibi arama motorlarında daha iyi sıralama elde etmesini sağlar.

On-Page SEO Temel Pratiği

Meta Başlık (Title): 50-60 karakter, anahtar kelime içermeli
Meta Açıklama (Description): 150-160 karakter, sayfanın içeriğini özetlemeli
Başlıklar (H1, H2, H3): Hiyerarşik yapı, H1 sayfa başlığı olmalıdır
URL Yapısı: Kısa, açıklayıcı, anahtar kelime içeren URL'ler tercih edin
İç Linkler: Sayfalar arasında anlamlı bağlantılar oluşturun
Görsel Optimizasyonu: Kompakt dosya boyutu, alt metin, responsive

SEO Meta Etiketleri Örneği

HTML Head

<meta charset="UTF-8">
<meta name="viewport"
  content="width=device-width,
           initial-scale=1.0">
<meta name="description"
  content="İyi açıklamalar sayfa
  sıramasını iyileştirir">
<meta name="keywords"
  content="anahtar, kelimeler">
<title>Sayfa Başlığı - Site Adı
</title>

Önemli Noktalar

  • 📱 Responsive design gereklidir
  • ⚡ Sayfa yükleme hızı önemlidir
  • 🔗 Dış linkler referans vermelidir
  • 📝 Kaliteli içerik yazın
  • 🔄 Yapı İşaretlemesi (Schema) kullanın
  • 🛡️ HTTPS kullanın
💡 İpucu: Google Search Console ve Analytics kullanarak sitenizin performansını izleyin ve iyileştirme fırsatlarını bulun.

5. Modern CSS Özellikleri

Son yıllarda CSS çok gelişti. İşte en önemli yeni özellikleri:

Container Queries

Container queries, bir elemanın ekran boyutuna değil, ana konteynerine göre responsive olmasını sağlar.

@container (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}

:has() Seçicisi

Bir elemanın çocuklarına göre stil uygulamayı sağlar (parent selector):

/* Eğer p elemanı içeriyorsa div'i renklendir */
div:has(> p) {
  border: 2px solid var(--primary);
}

/* Eğer hovered input içeriyorsa form'u renklendir */
form:has(input:hover) {
  background: rgba(99,102,241,0.05);
}

clamp() Fonksiyonu

Flexible sizing için kullanılır - minimum, tercih edilen ve maximum değerler belirtir:

/* Font boyutu: en az 16px, ideal 2.5vw, en fazla 32px */
h1 {
  font-size: clamp(16px, 2.5vw, 32px);
}

/* Padding: en az 1rem, ideal 5%, en fazla 2rem */
section {
  padding: clamp(1rem, 5%, 2rem);
}

Modern CSS Avantajları:

⚡ Daha Az JavaScript: CSS bunu yapabildiğinde neden JS kullanıyorsunuz?
🎯 Daha Responsive: Gerçek responsive tasarım yapın
🚀 Performans: CSS native çalışır, daha hızlı
📱 Esneklik: Herhangi bir ekran boyutu için optimize edin
⚠️ Dikkat: Tarayıcı uyumluluğunu kontrol edin. Bazı yeni özellikler henüz tüm tarayıcılarda desteklenmeyebilir.

6. Performans Optimizasyonu

Hızlı yüklenen web siteleri daha iyi dönüşüm oranlarına ve SEO sıralamasına sahiptir.

Performans Kontrol Listesi

Optimizasyon Etki Zorluk
CSS/JS Minifikasyon Dosya Boyutu ↓ Kolay
Görsel Optimizasyonu Yükleme Hızı ↑ Orta
Lazy Loading İlk Yükleme ↑ Orta
Caching Stratejisi Tekrarlayan Ziyaret ↑ Zor
Critical CSS First Contentful Paint ↑ Orta
Code Splitting JS Boyutu ↓ Zor

Performans Metrikleri (Core Web Vitals)

Largest Contentful Paint (LCP)

En büyük içerik ögesinin yükleme süresi (hedef: 2.5s altı)

1.0s
First Input Delay (FID)

Kullanıcı etkileşiminden yanıt süresine kadar (hedef: 100ms altı)

50ms
Cumulative Layout Shift (CLS)

Beklenmeyen düzen kaymalarının toplamı (hedef: 0.1 altı)

0.05

Performans İpuçları

1. Görsel Optimizasyonu: WebP formatı kullanın, uygun boyut seçin, compress edin
2. Kritik Kaynakları Ön Yükle: <link rel="preload"> kullanarak önemli kaynakları ön yükleyin
3. Inline Kritik CSS: Sayfa başında gösterilen CSS'i HTML'e embed edin
4. Defer JavaScript: Async veya defer özelliğini kullanarak JS yükleme süresini azaltın
5. CDN Kullanın: Statik içeriği CDN'den sunarak latency azaltın
6. Gzip Sıkıştırması: Server tarafında sıkıştırma etkinleştirin
💡 İpucu: Google PageSpeed Insights ve Lighthouse araçlarını kullanarak sitenizi analiz edin ve iyileştirme önerileri alın.
🎯 Alıştırma: Bir web sitenizi Google PageSpeed Insights ile taratın, ekran görüntüsü alın ve en az 3 tane iyileştirme yapın.
